Cybersecurity Basics  
CIA Triad
1. Confidentiality
•	Ensures that sensitive data is accessible only to authorized users 
•	Prevents unauthorized access 
Methods:
•	Encryption 
•	Passwords 
•	Access control 
Example:
Bank details are visible only to the account holder.
2. Integrity
•	Ensures data is accurate and not altered 
•	Protects against unauthorized modification 
Methods:
•	Hashing 
•	Digital signatures 
•	Checksums 
Example:
Marks stored in a database should not be changed by hackers.
3. Availability
•	Ensures systems/data are available when needed 
•	Protects against downtime or disruptions 
Methods:
•	Backup systems 
•	Redundancy 
•	Load balancing 
Example:
Online banking should work 24/7 without interruption.
 Threat Types 
1. Phishing
•	Fake emails/messages to steal sensitive information 
•	Tricks users into clicking malicious links 
Example: Fake bank login page
2. Malware
•	Malicious software that harms systems 
Types include:
•	Virus 
•	Worm 
•	Trojan 

3. DDoS (Distributed Denial of Service)
•	Floods a server with traffic to crash it 
Result: Website becomes unavailable

4. SQL Injection
•	Attacker inserts malicious SQL queries into input fields 
•	Gains access to database 
Example:
' OR '1'='1
5. Brute Force Attack
•	Tries all possible passwords until correct one is found 
Defense:
•	Strong passwords 
•	Account lockouts 
6. Ransomware
•	Encrypts files and demands money to unlock 
Example: WannaCry attack
Attack Vectors
1. Social Engineering
•	Manipulating people to reveal confidential information 
Example:
•	Pretending to be IT support 
2. Wireless Attacks
•	Exploiting insecure Wi-Fi networks 
Examples:
•	Man-in-the-middle attack 
•	Fake Wi-Fi hotspots 

3. Insider Threats
•	Attacks from employees or trusted users 
Types:
•	Malicious insider 
•	Negligent employee
